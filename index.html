Sensor Selection & Rationale

A sensor is an electronic device that detects changes in the environment and converts them into electrical signals that a robot or microcontroller (like ESP32/Arduino) can understand.

To study and understand the purpose, selection, and working principles of the main sensors used in a fruit-picking robot. These sensors enable the robot to detect ripe fruits based on colour, follow a fixed path between plant rows, and avoid obstacles while navigating.

Why Do Robots Need Sensors?

Robots use sensors to avoid obstacles, detect objects, measure distance, sense color, follow lines, and detect movement. Without sensors, a robot is blind and cannot react to the environment.

The Three Essential Sensors

1. Colour Sensor (TCS34725)

Detecting Ripe Fruits

Purpose

Identifies the colour of fruits to determine which are ripe (yellow) and which are unripe (green). The picking mechanism activates only for yellow fruits.

Working Principle

Shines white light onto the fruit. Photodiodes with RGB filters measure reflected light.

  • High Red + High Green + Low Blue â†’ Yellow (Ripe)
  • Low Red + High Green + Low Blue â†’ Green (Unripe)

Reason for Selection

  • Detects wide range of colours via RGB photodiodes.
  • Digital frequency output is easy to read.
  • Works well under varying lighting (sunlight/indoor).

Voltage: 3V â€“ 5V

Range: 1â€“3 cm

2. IR Line Sensors (HW-201)

Following the Path

Uses two IR reflective sensors (Left/Right) to follow a black line. White surfaces reflect IR (Output 1), while black lines absorb IR (Output 0).

What are Microcontrollers?

A microcontroller is a small computer built on a single chip that is designed to control electronic devices automatically. It has everything needed inside it â€” processor, memory, and input/output pins â€” so it can read signals from sensors and then control things like motors, LEDs, displays, etc.

It works by continuously running the program stored inside it to perform a specific task, like:

  • Turning a fan on/off based on temperature.
  • Making a robot move when a sensor detects an object.
  • Controlling washing machine cycles.

What Is Inside a Microcontroller?

CPU (Processor)

The thinking part â€” runs your program.

Memory

  • Flash Memory: stores your code.
  • RAM: stores temporary data while running.

Input/Output Pins (GPIO)

Used to read sensors and control motors/LEDs.

Timers & PWM

For controlling motors and servos smoothly.

Communication

I2C, SPI, UART

Used to connect to sensors, displays, etc.

Power Regulation

Allows the chip to work at stable voltage (3.3V or 5V).

ROS2 Architecture

What is ROS2?
ROS2 is not a programming language, not an operating system, and not magic â€œrobot software.â€ Itâ€™s a middleware framework that makes multiple programs in a robot communicate and work together in a predictable, scalable way.

If you try to build a serious robot without something like ROS2, you end up with spaghetti code where every component talks to every other component directly â€” impossible to maintain and guaranteed to break as soon as the system grows. ROS2 fixes that problem.

Why ROS2 exists?

Robots need a lot of components running at the same time: camera input, lidar processing, mapping, localization, motion control, navigation, planning.

If each of these modules communicated manually, you'd drown in complexity. ROS2 creates standard communication patterns so each part can do its job without depending on how other parts are implemented.

This is why advanced robots use ROS2 instead of custom code hacks.

Sensor data is collected inside Edge Impulse (images, audio, motion, etc.).

2. Extract Features

The data is cleaned, labelled, and features are extracted (important patterns).

3. Train Model

A machine-learning model is trained automatically using that processed data.

4. Deploy

The trained model is exported and installed on the target device.

5. Inference

The device starts making predictions in real time â€” for example detecting gestures, recognizing keywords, spotting objects, or sensing abnormal vibration â€” without needing the internet.
